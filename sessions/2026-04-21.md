### Architecture decisions

**Self-host on existing Vercel vs bcampx Docker vs Shlink vs keep Short.io**: chose Vercel because (a) already on Pro plan so incremental cost is zero, (b) destinations live in git (reviewable, reversible, no dashboard), (c) no API keys to rotate, (d) "if the website is up, short links are up" — one fewer thing to monitor.

**Three-kind redirect schema** (`property` / `path` / `external`):
- `property` handles the bulk (slug + optional page, auto-appends `?access=<DOCS_ACCESS_KEY>`)
- `path` covers shared pages like Queenstown Insights without special-casing
- `external` covers the one non-docs link (34 Shotover car-park Google Drive)
- Single schema, simple lookup, typed const map. No DB, no dashboard.

**Route implementation**: Edge runtime, GET only, unknown slug → 404 (not a default landing — deliberate, no path-enumeration signal). Access key read from env at request time so rotation doesn't require touching the map.

### DNS + TLS cutover

- `bcampx.com` is on Cloudflare nameservers (NOT Vercel DNS, which is what `snowbirdhq.com` uses — mental model correction for the project).
- Used Claude-in-Chrome browser automation to edit the Cloudflare DNS record — Type CNAME → A, Target cname.short.io → 76.76.21.21, TTL 1hr → 1min for cutover safety.
- DNS propagated to all major resolvers (1.1.1.1, 8.8.8.8) within ~2 minutes. Very fast — Cloudflare's default TTL + anycast.
- Vercel TLS cert (Let's Encrypt) took ~2–3 minutes after DNS pointed at Vercel. During that gap, HTTPS failed with `SSL_ERROR_SYSCALL` but HTTP-port-80 already worked correctly — proof the rewrite was live even before HTTPS was usable.
- Full end-to-end verification over live `https://go.bcampx.com`: all 10 sampled paths returned correct 302s, unknown slug returned 404, external (Google Drive) redirect worked, full browser flow (short link → docs → cookie set) passed.

### Vercel domain attachment quirks
