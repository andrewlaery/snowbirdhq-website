## Access model

Two HMAC-signed cookies, both signed with `DOCS_COOKIE_SECRET` (Vercel `snowbirdhq` production env):

| Cookie | Set by | Grants |
|---|---|---|
| `docs_property` | Short-link handshake — `?access=<DOCS_ACCESS_KEY>` on a `/docs/properties/{slug}` URL sets a slug-scoped cookie | The guest's own property + Queenstown Insights |
| `docs_portal` | Password form at `/access` — POST to `/api/access-unlock` with `DOCS_PORTAL_PASSWORD` | Full portal: `/`, `/properties`, every property |

Middleware (`src/middleware.ts`) verifies the HMAC on every request. `/docs/internal/*` and `/docs/owner-docs/*` hard-404 regardless of cookie.

## Short-link infrastructure (self-hosted, 2026-04-21)

`go.bcampx.com` serves from this repo — Short.io was retired on 2026-04-21. Links map lives at `src/lib/short-links.ts`; route handler at `src/app/s/[slug]/route.ts`. Destinations for docs pages append `?access=<DOCS_ACCESS_KEY>` at request time from env, so rotating the key only requires redeploying — no bulk short-link update.

## Access-key rotation

12-month cadence. Last rotated: 2026-04-20. Next rotation due: 2027-04-20.

1. Generate a new key: `openssl rand -hex 16`
2. Update the Vercel env var: `printf '%s' "$KEY" | vercel env add DOCS_ACCESS_KEY production` (use `printf`, not `echo`)
3. Redeploy (`vercel redeploy https://snowbirdhq.com` or `vercel --prod`) so the edge runtime picks up the new value

Cookies set with the old key become invalid at step 3. Guests re-enter through the same short link (which now appends the new key); new cookie is set and they continue. No short-link changes needed since the map doesn't hard-code the key.

## Deprecated per-property JWT scheme

The previous scheme signed per-slug guest tokens with `GUEST_TOKEN_SECRET`. It was retired on 2026-04-20 while the upstream Supabase project is rebuilt. `scripts/generate-guest-token.ts` and the `GUEST_TOKEN_SECRET` env var are left in place so a future session can restore that scheme without re-implementing it.
