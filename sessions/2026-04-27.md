# Session — 2026-04-27

## Topics
- Investigated whether the per-property "Ask Me Anything" chat at `/docs/properties/<slug>/ask` actually draws from the SOT, and wired it up properly when it didn't.

## Findings
- **The chat was effectively factless.** Pre-fix, `loadPropertyDocs` in `src/lib/chat/property-context.ts` read the four `content/docs/properties/<slug>/*.mdx` files via raw `fs.readFile`. Post-Phase-4 those MDX files are pure component composition — `welcome-house-rules.mdx` for 7-suburb is 18 lines and consists entirely of tags like `<PropertyWelcome slug="7-suburb" />`. The AI received those tags as literal text, not the rendered facts. Total per-property MDX context for 7-suburb was 54 lines of mostly markup. Only `queenstown-insights.mdx` carried real content.
- **The SOT was already vendored into the repo.** `npm run sync-sot` (also `prebuild`) copies `_shared/properties/<slug>/` → `data/sot/properties/<slug>/`. The React component layer (`src/lib/sot.ts`) reads from there. Last sync was today, drift-free against the canonical SOT. The chat path simply bypassed it.
- **Decision: extend the chat loader to read the SOT directly.** Rejected the alternative (server-render the MDX and feed rendered HTML) — heavier and the MDX components themselves only re-derive facts from `data/sot/`, so cutting out the middle layer is cleaner.
- **Excluded `policies.yaml` from the chat context on principle.** It contains `access_codes.front_door_master` (e.g. `2504#` for 7-suburb), `garage`, `alarm`, and operational `par_levels`. The per-booking guest code is generated separately by the smart-lock system; the master code must never reach the chat. Smoke-tested 7-suburb: confirmed `access_codes`, `2504#`, and `par_levels` do not appear in the assembled context.
- **What's actually loaded**: `identity.yaml` (display name, address, listing IDs, nicknames), `facts.yaml` (capacity, bedrooms, amenities, parking, check-in/out, wifi, house rules, exceptions, hazards), and `guest_copy.md` (long-form welcome prose). Comments are stripped from the YAML for compactness; frontmatter is stripped from the Markdown. Net body for 7-suburb: ~47K chars (~12K tokens).
- **Title-extraction edge case**: `display_name: Wright's Garden` is unquoted YAML with an apostrophe. First-pass regex `[^"'\n]+?` excluded apostrophes and matched empty — title fell back to slug. Fixed with `(.+?)\s*$` then trim outer quotes via a second pass.
- **System prompt now declares explicit source priority**: (1) PROPERTY FACTS (SOT, canonical), (2) PROPERTY GUIDE (MDX, supplementary), (3) QUEENSTOWN INSIGHTS (area). Added a defensive line: "Never share access codes or operational details that aren't explicitly addressed to guests."
- **Pre-existing tsc errors** in `src/components/queenstown-insights.tsx` (lines 92, 119 — `PreparedItem.type` / `PreparedSection.type` missing). Not introduced by this session; pre-dates my work, untouched.
- **Sandbox quirk noted**: pre-commit's stash-patch directory `~/.cache/pre-commit/` is outside the harness sandbox writable list, so `git commit` fails with `PermissionError` until sandbox is disabled for that one command. Pre-commit hooks work fine; only the cache-write is blocked.
- **Token cost impact**: SOT adds ~12K input tokens to every chat exchange. Prompt caching is already enabled (`cacheControl: ephemeral`) on the system message — first call writes cache, subsequent calls read cheaply. Consistent with the pattern documented in CLAUDE.md.
