## Current access scheme

A single shared `DOCS_ACCESS_KEY` stored in the Vercel `snowbirdhq` production env gates `/docs/properties/*` and `/docs/queenstown-insights`. Short.io `-010-LandingPage` destinations append `?access=<KEY>`. On first hit, middleware sets a one-year `docs_access` cookie on `docs.snowbirdhq.com` and redirects to the clean URL. `/docs/internal/*` and `/docs/owner-docs/*` are hard-404'd regardless of key until full per-role auth returns.

## Key rotation

12-month cadence. Last rotated: 2026-04-20. Next rotation due: 2027-04-20.

Rotating means:
1. Generate a new key: `openssl rand -hex 16`
2. Update the Vercel env var: `printf '%s' "$KEY" | vercel env add DOCS_ACCESS_KEY production` (use `printf`, not `echo`, to avoid trailing-newline corruption)
3. Redeploy (`vercel redeploy https://snowbirdhq.com`) so the middleware picks up the new value
4. Update the destination URL on each Short.io `-010-LandingPage` link

Old keys stop working at step 3. Cookies set with the old key become invalid at the same instant. Guests who bookmarked the clean URL will see a 404 until they re-open the Short.io link.

## Deprecated per-property JWT scheme

The previous scheme signed per-slug guest tokens with `GUEST_TOKEN_SECRET`. It was retired on 2026-04-20 while the upstream Supabase project is rebuilt. `scripts/generate-guest-token.ts` and the `GUEST_TOKEN_SECRET` env var are left in place so a future session can restore that scheme without re-implementing it.
