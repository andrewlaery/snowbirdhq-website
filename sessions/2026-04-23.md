## Findings

### Vercel AI SDK v6 is materially different from v4/v5

- `useChat` has moved from `ai` into `@ai-sdk/react` — separate install, separate import. v4 muscle memory (`import { useChat } from 'ai/react'`) gives a broken import silently.
- `convertToModelMessages()` is **async** now. Forgetting to `await` it gives a TS2488 ("must have a `[Symbol.iterator]()` method") error when you try to spread the result.
- Cache fields on the usage object moved to `usage.inputTokenDetails.cacheReadTokens` / `cacheWriteTokens`. The top-level `cachedInputTokens` is deprecated.
- Cache control on the system prompt MUST go via a `role: 'system'` message inside the `messages` array with `providerOptions.anthropic.cacheControl: { type: 'ephemeral' }` — the top-level `system` field is a plain string that won't accept providerOptions. Missed this in the first iteration.
- Streaming response helper renamed: `result.toUIMessageStreamResponse()` (v6), was `toDataStreamResponse()` in v4.
- UIMessage has a `parts[]` array shaped like `{ type: 'text'; text: string } | ...`, not a single `content` string. Filter + concatenate to render.

Saved all of the above as a new `## Vercel AI SDK v6` section in global MEMORY.md for next time.

### API routes bypass the docs-subdomain middleware by design

`next.config.mjs` rewrites `docs.snowbirdhq.com/*` → `/docs/*`, but explicitly excludes `api`, `auth`, `_next`, `icon`, `access` via a negative lookahead (`/:path((?!auth|api|_next|icon|access).+)`). That means `docs.snowbirdhq.com/api/chat/{slug}` resolves as `/api/chat/{slug}` — the docs middleware never runs. `/api/chat/[slug]/route.ts` therefore owns its own auth check: inline `verifyCookie` against `docs_portal` / `docs_property` cookies using the same helper the middleware uses. Fail-open when `DOCS_COOKIE_SECRET` is missing, mirroring middleware behaviour for local-dev ergonomics.

### Haiku 4.5 minimum cacheable prefix is 4096 tokens

Per the claude-api skill's cache reference table. Property docs at ~5K tokens are right at the edge. Adding Queenstown Insights (~7K) pushed total context to ~12K, well clear of the threshold. If the scope were ever shrunk below 4K, caching would silently fail (`cache_creation_input_tokens: 0` with no error) and per-message cost would ~10× — pad or accept the cost.

### Markdown rendering is a separate layer

Initial v1 shipped assistant messages as plain `<div>{text}</div>` with `whiteSpace: pre-wrap`. Claude's default answers use markdown heavily (bold for names, bullets for steps, headings for sections). Added `react-markdown` with a custom `components` map that applies the editorial-luxe design tokens (Newsreader serif for h1/h2, Geist for body, deep-teal for links, JetBrains Mono in a cream pill for inline code). User messages kept as plain text — they're echoing what the user typed, no markdown expected.

### Pre-commit + gitleaks hook plays nice with sandbox

The hook writes a stash patch to `~/.cache/pre-commit/`, which is outside the sandbox allowlist. Needs `dangerouslyDisableSandbox: true` for commits when there are unstaged files in the working tree.
