# Internationalisation (i18n) — Simplified Chinese pilot

Pilot live as of 2026-04-30 on `7-suburb` only. Designed as a deletable narrow-route experiment, **not** a full Fumadocs i18n setup.

## Architecture choice

Narrow `/zh/*` route subtree, NOT full Fumadocs i18n. Rationale: full Fumadocs i18n requires restructuring every existing route (`[lang]/` segment in `source.config.ts`, `loader()` config, content-tree restructure for all 13 properties) — wrong cost for a single-property test. The `/zh/*` subtree at `src/app/zh/` is self-contained: deleting the whole tree reverts the pilot.

## Routing

`next.config.mjs` rewrite carves `/zh/*` out of the `docs.snowbirdhq.com` catch-all (regex `(?!auth|api|_next|icon|access|zh)`) so `/zh/*` URLs hit `src/app/zh/*` directly without the `/docs/` prefix. EN routes unaffected.

## Translation pipeline

`scripts/translate-property.mjs` — CLI takes a slug + lang, calls **Claude Sonnet 4.6** to translate `data/sot/properties/<slug>/{identity.yaml, facts.yaml, guest_copy.md}` into `data/sot/properties/<slug>/<lang>/`. SHA-256 hashes recorded in `.translation-manifest.json`; re-runs skip files whose source hasn't changed. Cost: ~$0.36 one-time per property (~14k input + 19k output tokens), ~$0.05–0.10 per refresh.

## SOT loader extension

`src/lib/sot.ts` `loadIdentity` / `loadFacts` / `loadProse` now accept an optional `lang: 'en' | 'zh'` parameter (default `'en'`, non-breaking for EN call sites). When `'zh'`, `slugDir` resolves to `data/sot/properties/<slug>/zh/`.

## Why no extractor changes for ZH

The existing `extractSections` in `sot.ts` only matches H2 headings, but `guest_copy.md` uses H1 for top-level sections. ZH pages bypass that with `src/lib/zh-guest-copy.ts` — a small H1-slicer that returns the body under a named H1 heading. (See backlog: `PropertyWelcome` is silently broken on EN for the same reason.)

## Why no fork of `HouseRulesBase` / `CriticalInfoBase` / `QueenstownEssentials`

Those long hard-coded English JSX components have direct content equivalents inside `guest_copy.md` (House Rules, Critical Information, Essentials sections). The ZH pages render the translated guest_copy.md sections directly via `react-markdown` instead of forking ~3000+ words of JSX.

## AMA in Chinese

`/api/chat/[slug]?lang=zh` reads the lang query param and switches the system-prompt language directive (`Reply EXCLUSIVELY in Simplified Chinese (简体中文)` etc.). **Property context stays English** — Haiku 4.5 reads EN fluently and outputs ZH; this avoids doubling context tokens and keeps the EN SOT as the single source of truth. Cache key is now `${slug}:${lang}` to avoid cross-locale poisoning. UI strings (suggestion chips, placeholder, error, "Thinking…") are forked into `src/components/property-ask-chat-zh.tsx` with native Chinese equivalents.

## LocaleSwitcher

`src/components/locale-switcher.tsx` — fixed-position pill (top-right) added to `src/app/docs/layout.tsx` (EN side) and inline in the header of `src/app/zh/layout.tsx` (ZH side). The Fumadocs `nav.children` slot was tried first but renders the children jammed against the logo with characters wrapping vertically — fixed-position float was the workaround. Switch is URL-only (no cookie persistence, no `Accept-Language` detection); the user clicks the pill or edits the URL.

## Files

`scripts/translate-property.mjs`, `data/sot/properties/<slug>/zh/`, `src/app/zh/layout.tsx`, `src/app/zh/properties/[slug]/{page,welcome-house-rules/page,critical-info/page,user-instructions/page,ask/page}.tsx`, `src/components/{locale-switcher,property-ask-chat-zh,property-quick-info-zh}.tsx`, `src/lib/zh-guest-copy.ts`, plus `lang` param threaded through `src/lib/sot.ts`, `src/lib/chat/{prompt,sot-context}.ts`, `src/app/api/chat/[slug]/route.ts`, and the `next.config.mjs` rewrite carve-out.

## Known gaps

- **Middleware gating on `/zh/*`**: `src/middleware.ts` enforces the property/portal cookie gate on `/docs/properties/*` but **not** on `/zh/properties/*`. The ZH pilot pages are currently ungated on `docs.snowbirdhq.com`. Acceptable for a 1-property pilot, but the AMA API (`/api/chat/[slug]`) IS gated, so a ZH visitor who lands cold can read the docs but the chat returns 401 until they get a portal/property cookie via the existing flow. Fix before promoting beyond the pilot: extend the Short.io property-cookie handshake to also work on `/zh/properties/<slug>` URLs.
- **`<html lang>`**: ZH pages inherit `<html lang="en">` from the root layout. Cleanly fixing this requires a Next.js route-group restructure (`src/app/(en)/`, `src/app/(zh)/`). `hreflang` link tags ARE set in metadata, so SEO discoverability works.

## Promotion path

When a 2nd property needs Chinese (or any other locale), promote to full Fumadocs i18n: add `[lang]` segment to all docs routes, configure `i18n` in `source.config.ts` and `loader()`, restructure `content/docs/` tree, retire `src/app/zh/` and the inline forks. The translation script and `lang`-aware SOT loader carry over unchanged.
